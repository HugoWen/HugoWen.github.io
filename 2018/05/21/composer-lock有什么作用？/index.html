Composer 背景

Composer 是 PHP 项目的依赖管理器，用来解决PHP项目没有快速引入三方库代码的情况。比较类似 Node 的 NPM。

Composer 组成

composer.json：项目安装文件
初始化文件，在文件内定义需要引入的依赖关系，包括包名和版本号。

composer.lock：锁文件
锁定项目中引入的三方依赖的报名和版本号。

composer.lock 有什么作用？

在安装三方依赖之后，Composer会把安装时确切的包名和版本号写入 composer.lock 文件，起到锁定版本号的的作用，下一次执行 composer install 的时候，将会直接读取 composer.lock 文件，并使用其中写入的指定版本号，不会尝试去拿更新或者其他版本的三方依赖。

这对于团队开发或者多人共同维护的项目比较有用处，可以保证各人之间使用的依赖都是同一版本的，也可以保证服务器与本地使用的也是同一版本，这样就不会有因为版本差异而带来的未知问题。所以这也要求，要将 composer.lock 文件上传到代码版本库中。

上边也有讲到，当执行 composer install 的时候会直接读取 composer.lock 内的内容，并不会进行修改。假如不存在 composer.lock 文件，会根据 composer.json 安装依赖，安装好依赖之后会自动根据当前的版本依赖创建一份 composer.lock 文件。

当执行 composer update 的时候，会同时更新 composer.lock 文件到最新的依赖版本号。
